## 任务一.初探网站设计

#### 一.网站和网页的概念`P176`

##### 1.网站

网站是指在互联网上，根据一定的规则，使用HTML语言等工具制作的用于展示特定内容的相关**网页的集合。**

##### 2.网页

网页是构成网站的基本元素，是承载各种网站应用的平台。网页实际上是一系列文件，通过网址来识别与存取

##### 3.网站的分类

- 按照提供的内容：咨询类网站，交易类网站，体验类网站，展示类网站，综合类网站
- 按照主题性质：政府网站，企业网站，商业网站，教育科研机构网站，个人网站，非盈利机构网站以及其他类型网站

##### 4.网页的分类

一般分为静态网页和动态网页

**静态网页**不能简单地理解成静止不动的网页，它主要指的是网页中没有程序代码，**只有HTML**（即：超文本标记语言），一般后缀为**`.html,.htm`**等。虽然静态网页的页面一旦做成，内容就不会再改变了。但是，静态网页也包括一些能动的部分，这些主要是一些`GIF`动画等。静态网页的打开，用户可以直接双击，并且不管任何人任何时间打开的页面的内容都是不变的。

**动态网页**是指跟静态网页相对的一种网页编程技术。动态网页的网页文件中除了HTML标记以外，还包括一些特定功能的程序代码，这些代码可以使得浏览器和服务器可以交互，所以服务器端根据客户的不同请求动态的生成网页内容。

即：动态网页相对于静态网页来说，页面代码虽然没有变，但是显示的内容却是可以随着时间、环境或者数据库操作的结果而发生改变的。动态网页，与网页上的各种动画、滚动字幕等视觉上的动态效果没有直接关系，动态网页也可以是纯文字内容的，也可以是包含各种动画的内容，这些只是网页具体内容的表现形式，**无论网页是否具有动态效果**，只要是采用了动态网站技术（如**`PHP、ASP、ASPX、JSP`**等）生成的网页都可以称为动态网页。

作用：是构成 Web 世界的一砖一瓦，定义了网页内容的含义和结构。可以用于创建网页等超文本文档。

## 任务二.制作简单网页

#### 一.HTML文档基本结构`P183`

1.以下是一段`HTML5`代码的基本结构

```html
<!DOCTYPE html>
<html>
	<head>
		<meta charset="UTF-8">      用于字符编码声明，确保网页内容以UTF-8格式编码
		<title>网页标题</title>
	</head>
	<body>

	</body>
</html>
```

- **`<!DOCTYPE html>`**声明了文档类型，是`HTML5`文档的标准声明，不区分大小写。

- ***`<html>`*** 标签定义了整个HTML文档的范围，是所有其他元素的父元素，确保浏览器能够正确解析和渲染。

- ***`<head>`*** 包含了文档的元数据，如字符编码声明、页面标题、引入`CSS`样式和JavaScript脚本等。

  ​                      					head中**常用的标签**以及内容

  |    标签    | 作用                                               |
  | :--------: | :------------------------------------------------- |
  |  `<meta>`  | 定义字符编码，视口（`Viewport`）设置，页面描述等   |
  | `<title>`  | 定义页面的标题，显示在浏览器标签页和搜索引擎结果中 |
  |  `<link>`  | 用于链接外部资源，如`CSS`样式表                    |
  | `<script>` | 用于嵌入或引入JavaScript脚本                       |

- *<body>* 包含了用户能在浏览器中看到的所有内容，如文本、图片、视频、表格等。

#### 二.HTML标签

##### 1.`HTML`标签的概念`P184`

HTML标签是HTML文档中的基本组成单位，负责定义文档的结构和内容，标签以尖括号< 和 >包裹，分为开始标签和结束标签。

##### 2.`HTML`标签的类型`P185`

- 成对标签（Paired Tags）

  成对标签包括一个开始标签和一个结束标签，元素内容位于这两个标签之内。示例如下：

  | `**<p>...</p>**` |   **定义段落**   |
  | :------------------------: | :--------------: |
  |      `<div>...</div>`      | 定义一个通用容器 |
  |  `<h1>...</h1>`  |   定义一级标题   |

- ###### 自闭合标签（Void Tags）

  自闭合标签不需要结束标签，通常用于插入内容或执行特定功能。示例如下：

  | `<img>`  |  插入图片  |
  | :------: | :--------: |
  |  `<br>`  |    换行    |
  |  `<hr>`  | 水平分割线 |
  | `<meta>` | 定义元数据 |

##### 3.`HTML`元素`P185`

- HTML 元素概念

  HTML元素是由HTML标签及其内容组成的整体，**是构成HTML文档的基本单元**。

- HTML 元素的常见格式

  ​      <标签名称 属性 1=“属性值” 属性 2=“属性值”.... >元素内容</标签名称>

  ​      例如：`<p id="main" class="text">这是一个段落。</p>`

  ​      `<p>`是开始标签，定义了一个段落元素，标签名是p      

  ​      ID 和class分别是属性，“main”和“text”分别是属性的值，为该元素提供了唯一标识

  ​      “这是一个段落。”是元素内容

  ​      `</p>`这是结束标签，标记段落内容的结束。

  ​	HTML标签和属性通常不区分大小写

- 常用的HTML元素

  使用基本的标签元素有助于帮助开发者清晰的组织信息

  | 结构元素 | 确保网页在不同设备和浏览器上正确显示             |
  | :------: | ------------------------------------------------ |
  | `<html>` | 定义HTML文档的根元素                             |
  | `<head>` | 包含文档的元数据（如标题，字符编码，样式链接等） |
  | `<body>` | 包含网页的可见内容（如文本，图片，链接等）       |

  |   文本内容元素   | 用于组织，格式化文本，并赋予特定的语义               |
  | :--------------: | ---------------------------------------------------- |
  |      `<p>`       | 定义段落，可以单独使用，作用是标记后面的文字换段显示 |
  |   `<h1>~<h6>`    | 定义文档的各级标题。1号最大，6号最小                 |
  | `<strong>和<em>` | 分别用于加粗和斜体                                   |
  |     `<span>`     | 用于设置文本内联样式或脚本操作                       |

  | 多媒体元素 | 多媒体元素用于嵌入和管理网页中的多媒体内容  |
  | :--------: | ------------------------------------------- |
  |  `<img>`   | 嵌入图片，图片格式可以是**`GIF，JPG和PNG`** |
  | `<audio>`  | 嵌入音频文件                                |
  | `<video>`  | 嵌入视频文件                                |

  | 链接元素 | 链接元素用于在网页之间或网页内部创建超链接                   |
  | :------: | ------------------------------------------------------------ |
  |  `<a>`   | 定义超链接，可以通过文本，图像，或者热点区域等链接到其他目标 |
  | `<nav>`  | 定义导航链接的集合，通常用于网站的导航栏                     |

在HTML中，注释用于在代码中添加说明性文字，这些文字**不会显示在网页上**，但是可以帮助开发者更好的理解和维护代码。例如：

## 任务三.修饰页面元素

#### 一.`CSS`规则`P196`

`CSS样式`表由一系列规则组成，每条规则包括选择器和声明模块。其中基本语法如下：

```css
选择器 {
	属性1：属性值;
	属性2：属性值;
}
例如:定义h1的样式表：文本颜色为红色，字体大小为14px
h1{
    color:red;
    font-size:14px;
}
```

在`CSS`中注释以`/*`开头，以`*/`结尾，中间的内容不会被浏览器解析或显示，对页面最终呈现效果没有直接影响

#### 二.常见的选择器

选择器是`CSS规则`的重要组成部分，通过精确选择目标元素，以便为其应用样式。常见的有以下分类

1. 标签选择器`P196`

   直接匹配某类HTML标签，作用于页面中所有该类型元素。范围广，适合定义通用样式

   ```css
   标签名{
       属性 1:属性值 ;
       属性 2:属性值 ;
   }
   ```

2. 类选择器`P197`

   通过HTML元素的class属性选择具有特定类名的元素。可作用于**多个元素**，**一个元素也可以有多个类**，灵活且可复用，推荐作为主要样式绑定方式。

   ```css
   .类名称{
       属性 1:属性值 ;
       属性 2:属性值 ;
   }
   ```

3. id选择器`P197`

   通过HTML元素的id属性选择具有特定id的元素。仅匹配页面中的唯一的元素，精确度高，优先级要高于类选择器，通常用于唯一性样式或js绑定。

   ```css
   #id{
       属性 1:属性值 ;
       属性 2:属性值 ;
   }
   ```

4. 后代选择器`P198`

   后代选择器是组合选择器之一，用于选择某个父元素中的所有后代元素，父元素与后代元素之间用空格分隔

   ```css
   父元素 后代元素{
       属性 1:属性值 ;
       属性 2:属性值 ;
   }
   ```

5. 伪类选择器`P198`

   用于选择元素的特定状态或位置。当元素处于某个特定状态时，它被应用伪类样式表；当状态改变时它又会失去这个样式

   ```css
   选择器:伪类{
       属性 1:属性值 ;
       属性 2:属性值 ;
   }
   ```

   以下是`CSS`中用于控制超链接不同状态的四个伪类选择器

- `a:link`：定义链接在未被访问过时的样式
- `a:visited`：定义链接被访问过后的样式
- `a:hover`：定义光标悬停在元素上的样式。不仅可以用于`<a>`还可用于几乎所有的HTML元素
- `a:active`：定义元素被激活（如按下鼠标左键但是未被释放）时的样式

#### 四.`CSS`样式表的使用和优先级`P202`

在网页中应用`CSS样式`时,根据样式定义位置的不同存在三种方式:行内样式、内部样式和外部样式。一般优先级为：行内样式 > 内部样式 > 外部样式

#### 五.知识链接

##### 1.外部样式`P204`

外部样式表是`CSS`的一种应用方式，通过将`CSS`代码存储在一个以`.css`为扩展名的文件中，并在HTML文档中的`<head>`部分通过`<link>`标签引用该文件，从而实现对网页样式的集中管理。

1. 基本语法格式

   `<link href="样式表文件路径" type="text/css" rel="stylesheet">`

   - `<link>`标记必须放在`<head>`和`</head>`标签内
   - `href` ：定义链接的样式表文件路径
   - `type` ：定义所链接的文档的类型，**“`text/css`”表示为`CSS样式表`**
   - `rel` ：定义当前文件与被链接文档的关系，**`stylesheet`代表这是一个样式文件**

2. 特点

   - 可重性：一个外部表可以被多个HTML页面引用，方便在多个页面间共享样式
   - 结构清晰：将样式表和HTML结构分开，有助于提升代码可读性和维护性
   - 易于维护：修改外部样式表可以一次性影响所有引用该样式表的页面

##### 2.内部样式`P209`

1. 基本语法结构

   ```html
   <head>
       <style type="text/CSS">
           选择器 {属性1:属性值1; 属性2:属性值2; 属性3:属性值3;}
       </style>
   </head>
   ```

2. 特点

   - 局部性：内部样式仅对当前HTML页面生效，不会影响其他页面内容
   - 嵌入性：`CSS`规则嵌入在HTML文件中，使文件结构较为紧凑，但随着样式复杂程度增加，可能会导致HTML文件难以维护

##### 3.行内样式`P213`

行内样式是将`CSS`样式直接写在HTML元素的style属性中，**只对当前元素生效**

1. 基本语法格式

   ```html
   <标签名 style="属性 1: 属性值;属性 2: 属性值;属性 3: 属性值………">内容</标签名>
   ```

2. 特点

   - 局部控制：行内样式只影响特定的HTML元素，适合对单个元素进行样式调整
   - 优先级高：行内样式的优先级高于外部和内部，因此它可以覆盖前两者的设置
   - 可读性差：行内样式使HTML代码变得冗长,难以维护

##### 4,`CSS`样式特性`P216`

1. 层叠性

   层叠性是指作用在一个元素上多种`CSS`样式的叠加。

2. 继承性

   继承性就是指定义`CSS`样式时,通常子元素中与文本相关的属性会继承父元素的值。

3. 优先级

   `CSS选择器`的优先级决定了当多个规则应用于同一元素时,哪条规则会生效。

- 行内样式:权重1000

- ID选择器:权重100

- 类选择器、属性选择器、伪类选择器:权重10

- 标签选择器、伪元素选择器:权重1

- 通配符选择器、继承的样式:权重0

  选择器权重越高,优先级越高;如果权重相同,则后定义的样式会覆盖前面的样式(就近原则:如果权重相同且声明顺序相同,则!important 声明优先。

## 任务四.`DIV+CSS`布局

#### 一.盒子模型的组成`P220`

HTML中的大多数元素都可以被视为包裹在一个矩形盒子内，这个盒子称为盒模型（Box Model）

它拥有盒子一样的外形和平面空间，从内到外依次由内容（`content`），内边距（`padding`），边框（`border`），外边距（`margin`）四个部分组成。

![](.\img\boxmodle.png)

1. 内容（Content）`P220`

   这是元素的实际内容区域,用于显示实际内容,如文本、图像等,如上图橘色部分所示。其尺寸可以通过 width 和 height属性来设置。

2. 内边距（Padding）`P220`

   位于内容区域与边框之间的空间,如上图中黄色部分所示。设置 padding属性,可以控制元素的内边距。

   如果要设置元素上、右、下、左四个方向的内边距,则可以设置 padding-top、padding-right、padding-bottom 和 padding-left 这四个属性。
   为了简化代码,可以直接设置`padding`属性,根据参数值个数不同有以下方法：

   -  提供四个参数值,将默认依次按照**上、右、下、左**四个方向设置元素的内边距例如
   - 提供三个参数值,则第一个值为**上**内边距,第二个值为**右、左**内边距,第三个值为**下**内边距
   - 提供两个参数值，第一个值为**上下**内边距，第二个值为**左右**内边距
   - 提供一个参数，表示**上下左右**内边距均为该值

3. 边框（Border）`P221`

   边框是围绕内容和内边距的线条,它是内边距和外边距的分界线

   使用以下属性可以分别设置元素的上、右、下、左四个边框,每个属性均可以设置宽度、样式和颜色,语法与 border 属性相同。

   | 属性            | 描述       |
   | --------------- | ---------- |
   | `border-top`    | 设置上边框 |
   | `border-right`  | 设置右边框 |
   | `border-bottom` | 设置下边框 |
   | `border-left`   | 设置左边框 |

4. 外边距（Margin）`P222`

   位于元素的外部空间,如上图中的绿色部分所示。设置 margin 属性,可以控制元之间的距离,调整页面布局,使内容更加整齐有序。与内边距类似,分别设置`margin-top、margin-right、margin-bottom 和 margin-left` 这四个属性可以独立控制元素四个方向的外边距。数值个数表示方法与内边距通用

#### 二.盒子模型尺寸`P223`

​	在`CSS`中,width 和 height属性专门用于定义元素内容区域(content)的宽度和高度,两个属性所设定的尺寸并不包含元素的外边距(margin)、边框(border)和内边距（padding）如果盒中没有内容,即使定义了高度和宽度都为100%,实际上元素也不会显示出来，这是因为内容区域的高度和宽度是基于其内容来计算的,如果没有,则内容区域宽度和宽度为0。

- 总宽度=**内容区域宽度(width)**+左内边距(padding-left)+右内边距(padding-right)+左边框(border-left)+右边框(border-right)+左外边距(margin-left)+右外边距(margin-right)

  同理元素的总高度也通循类似的计算方式。

- 总高度=**内容区域高度(height)**+上内边距(padding-top)+下内边距(padding-bottom)+上边框(border-top)+下边框(border-bottom)+上外边距(margin-top)+下外边距(margin-bottom)

#### 三.`<div>`元素`P224`

在HTML和`CSS`中,`<div>`元素是一个没有特定语义的块级容器元素,它经常用其他 HTML元素并进行布局。以下是`<div>`作为块状元素的一些主要特性

1. 块级元素

   `<div>`默认是一个块级元素，这意味着它从新的一行开始，占据其父容器的整个宽度

2. 默认显示特性

   `<div>`默认会在其后面添加换行，即在垂直方向上堆叠

3. 容器角色

   `<div>`经常作为其他元素的容器，可以包含文本，图片，链接，其他容器等

4. 可以嵌套

   `<div>`可以嵌套其他`<div>`元素，以及其他块级或行内元素，形成复杂的页面结构

5. 样式应用

   可以对`<div>`应用`CSS`样式，如背景颜色，边框，内边距，外边距等

6. 布局控制

   `<div>`可以通过`CSS`进行灵活的布局控制，如使用浮动（float），定位（position）等

#### 四.文档流`P224`

在网页设计中,文档流是指HTML文档中元素的默认排列方式。它是浏览器根据文档的结构和 `CSS`样式规则,以特定的顺序和逻辑将页面内容渲染并呈现给用户的核心机制。有以下两个特点

1. 从上到下，从左到右的排列

   在文档流中,块级元素(如`<div>、<p>、<h1>`等)会默认占据一行或多行,按照从上到下的顺序排列

   行内元素（如`<span>，<a>，<img>`等）则在同一行内从左到右排列，直到满行后执行

2. 元素的自然排列

   文档流中的元素会按照HTML代码中的顺序依次排列，不会发生重叠或交错

   在没有设置特定宽度值时，块级元素会默认占据其父元素的可用宽度，而行内元素的宽度和高度会根据其内容和`CSS`设置自动调整

#### 五.浮动属性`P226`

在网页设计中，浮动属性（float）是一种强大的`CSS`布局工具，用于控制元素的排列方式和位置。

1. 浮动属性的基本概念

   浮动属性可以使元素脱离文档流，并向左或右浮动。浮动的元素会尽可能地向指定方向移动，直到遇到容器边界或其他浮动元素为止。

   基本语法格式：选择器 { float : 属性值 }

   | 属性值  |          描述          |
   | :-----: | :--------------------: |
   | `left`  |     使元素向左浮动     |
   | `right` |     使元素向右浮动     |
   | `none`  | 使元素不浮动，为默认值 |

   

2. 最重要的总结

   `float:left` 应该理解为：**在自己本身的位置上脱离文档流，元素从包含框的右边缘向左移动，直到碰到包含框的左边缘或者其他浮动元素框停下，当空间不足以绘制自己则另起一行。**

   `float:right` 应该理解为：**在自己本身的位置上脱离文档流， 元素从包含框的左边缘向右移动，直到碰到包含框的右边缘或者其他浮动元素的框停下，当空间不足以绘制自己则另起一行。**

3. 清除浮动属性`P231`

   因为浮动元素会脱离正常文档流，可能会对后续元素的布局产生影响，可能导致主题区域的高度无法自动撑开，进而出现**主体高度塌陷**。此时就需要清除浮动属性后才能按照正常的文档流进行排列。

   基本语法格式为：选择器 { clear : 属性值 }

   | 值      | 描述                             |
   | ------- | -------------------------------- |
   | `left`  | 清除左侧浮动元素的影响           |
   | `right` | 清除右侧浮动元素的影响           |
   | `both`  | 同时清除左，右两侧浮动元素的影响 |

#### 六.响应式布局`P232`

响应式布局是指网页能够自动适应不同屏幕尺寸，分辨率和方向，确保用户在任何设备上都能获得最佳的浏览体验。

1. 基本格式

   ```html
   @media (条件){
   	CSS-Code;       /* 当条件满足时应用的CSS规则 */   
   }
   ```

2. 常用媒体查询条件（以考*1）

   |         设备视口宽度         |                媒体查询条件                 |        描述        |
   | :--------------------------: | :-----------------------------------------: | :----------------: |
   |      视口宽度 ≤ `768px`      |             `max-width : 768px`             |   适用于手机设备   |
   | `769px` ≤ 视口宽度 ≤ `991px` | `(min-width:769px) and (max-width : 991px)` | 适用于平板电脑设备 |
   |      `992px` ≤ 视口宽度      |             `min-width : 992px`             |      适用于PC      |

3. 示例（以考*1）

   - 视口（`Viewport`）设置

     在HTML文档的`<head>`部分添加`<meta>`标签，对视口进行设置。

     ```html
     <meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">
     ```

     | 常用属性             | 描述                                                         |
     | -------------------- | ------------------------------------------------------------ |
     | `minimum-scale`      | 设置网页的最小缩放比例。比如`minimum-scale=0.5`，就表示网页最小只能缩小到原始大小的 0.5 倍。 |
     | `maximum-scale`      | 设置网页的最大缩放比例。比如`maximum-scale=2.0`，就表示网页最大只能放大到原始大小的 2.0 倍。 |
     | `user-scalable`      | 设置用户是否可以手动缩放网页。`user-scalable=no`表示用户不能手动缩放，`user-scalable=yes`表示用户可以手动缩放。 |
     | `name="viewport"`    | 告诉浏览器，这是一个视口设置的`<meta>`标签。                 |
     | `width=device-width` | 把视口的宽度设置为设备的宽度。这样，网页就能根据设备屏幕的实际宽度来显示，不会出现显示不全或者有多余滚动条的问题。 |
     | `initial-scale=1.0`  | 设置网页的初始缩放比例为 1.0。也就是说，网页加载的时候，不会自动缩放，而是按照原始大小显示。 |

   - 媒体查询

     媒体查询是 `CSS3` 引入的强大功能，它可以根据不同的设备屏幕特性（如宽度、高度、分辨率等）应用不同的 CSS 样式。通过在 CSS 中使用`@media`规则，你可以为不同的屏幕尺寸定义不同的样式。

     以下是一个简单的示例，展示了如何使用媒体查询来实现网页适配：

     ```html
     /* 当屏幕宽度小于等于 600px 时应用这些样式 */
     @media (max-width: 600px) {
         body {
             font-size: 14px;
         }
         .container {
             width: 100%;
             padding: 10px;
         }
     }
     
     /* 当屏幕宽度大于 600px 且小于等于 900px 时应用这些样式 */
     @media (min-width: 601px) and (max-width: 900px) {
         body {
             font-size: 16px;
         }
         .container {
             width: 80%;
             margin: 0 auto;
         }
     }
     
     /* 当屏幕宽度大于 900px 时应用这些样式 */
     @media (min-width: 901px) {
         body {
             font-size: 18px;
         }
         .container {
             width: 60%;
             margin: 0 auto;
         }
     }
     
     ```
